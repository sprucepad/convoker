Commands at the moment only supports POSIX-like argument parsing, but we plan to make this completely customizable in the future, to support Windows-like parsing and GNU-like parsing.

```sh
my-app \
  sub-command \
  sub-sub-command \
  --long-option=value \
  --long-bool-option \
  -b \
  -q=short-non-boolean-flag
```

Convoker also allows for middleware, functions that run before your action function is reached, in order, that have control flow.

## Usage

First, you create an instance of a `Command`, passing it a name to show on help screens. This will act as your root command, as the main entry point of your program. You can then further modify it using helper methods, like `description` or `version`, or `action` if you want your root command to do something.

```ts
import { Command } from "@convoker/core";

const program = new Command("demo")
  .description("A demo Convoker application.")
  .version("1.0.0");
```

You can then add subcommands, using three main methods:

1. Standard builder pattern, which only allows one subcommand per chain.

   ```ts
   program
     .subCommand("name")
     .use(middleware)
     .input(inputObj)
     .description("description")
     .version("0.1.0")
     .action(actionFn);
   ```

2. Callback function, which allows you to define multiple subcommands in the same chain.

   ```ts
   program.subCommand("name", (c) =>
     c
       .use(middleware)
       .input(inputObj)
       .description("description")
       .version("0.1.0")
       .action(actionFn),
   );
   ```

3. Imported, which allows you to define subcommands in different files.

   ```ts
   const name = new Command("name")
     .use(middleware)
     .input(inputObj)
     .description("description")
     .version("0.1.0")
     .action(actionFn);

   program.add(name);
   ```

Then, you can execute it.

```ts
program.run();
```

`program.run()` simply returns a promise, so you can do anything you want before or after it. Convoker has a default help screen, which you can override by changing the default error handler and checking for `HelpAskedError`.

Command action functions, error handlers, middlewares, any function, can take in an _input_ as the first argument, and middleware functions have a second parameter (`next`, which calls the next function in the chain). See [Input parsing](./input) for how to define inputs!
