title: Input parsing
description: How input parsing in commands work in Convoker.
---

Convoker allows type-safe and runtime-safe input definition for your commands. You can use a simple string, a validator function or any Standard Schema validator.

[The input module is tied to and comes with the core](./command).

## Usage

Input is used within commands. Commands have an `input` function, that allows you to pass an object containing your input, which you can then access in middleware or actions.

The input module is exposed as `i` from the core or bundle package.

```ts
import { Command, i } from "@convoker/core";
import { z } from "zod";

const program = new Command()
  .input({
    positionalArgument: i.positional(z.string()), // or any Standard Schema validator, such as Valibot, or a basic type.
    option: i.option("string", "--hello", "-h").optional(),
  })
  .action(({ positionalArgument, option }) => {
    console.log(positionalArgument, option);
  });
```

There are three main functions for defining inputs: `i.positional` and `i.argument` (which are aliases to each other), which define positional arguments, and `i.option`, which defines a single or multi-character option. Each take in a validator or type as the first argument, and options take in a variadic argument for the option names, which must start with `--` or `-`.

This input object is then parsed as CLI arguments, and is passed into middleware and action functions as the first argument.
